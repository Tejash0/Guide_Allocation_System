### Task 1: Backend — Add `?domain=` query param to GET /api/faculty/available

**File**: `backend/routes/faculty.js`

**What**: Extend the `/available` route (lines 57–69) to accept an optional `domain` query string parameter. When present, filter results with a case-insensitive `LIKE` match against the faculty's `domain` column. When absent, return all approved faculty (existing behavior unchanged).

**How**:

Replace lines 57–69 (the entire `router.get('/available', ...)` block) with:

```js
router.get('/available', (req, res) => {
  const { domain } = req.query;

  let rows;
  if (domain && domain.trim().length > 0) {
    rows = db.prepare(`
      SELECT
        f.id, f.name, f.department, f.domain, f.max_teams,
        COUNT(r.id) AS current_team_count
      FROM faculty f
      LEFT JOIN requests r ON r.faculty_id = f.id AND r.status = 'accepted'
      WHERE f.approved = 1
        AND LOWER(f.domain) LIKE LOWER(?)
      GROUP BY f.id
      ORDER BY f.name ASC
    `).all(`%${domain.trim()}%`);
  } else {
    rows = db.prepare(`
      SELECT
        f.id, f.name, f.department, f.domain, f.max_teams,
        COUNT(r.id) AS current_team_count
      FROM faculty f
      LEFT JOIN requests r ON r.faculty_id = f.id AND r.status = 'accepted'
      WHERE f.approved = 1
      GROUP BY f.id
      ORDER BY f.name ASC
    `).all();
  }

  res.json(rows);
});
```

**Verify**: `curl "http://localhost:3001/api/faculty/available?domain=ML"` returns only faculty whose `domain` contains "ML" (case-insensitive). `curl "http://localhost:3001/api/faculty/available"` returns all approved faculty.

### Task 2: Frontend API — Accept optional `domain` param in `getAvailableGuides()`

**File**: `frontend/src/api/faculty.js`

**What**: Update `getAvailableGuides()` (line 10–13) to accept an optional `domain` string. When provided, append `?domain=<value>` to the fetch URL. When omitted or empty, fetch without query string (preserving current behavior).

**How**:

Replace lines 10–13 with:

```js
export async function getAvailableGuides(domain) {
  const url = domain && domain.trim().length > 0
    ? `${BASE_URL}/available?domain=${encodeURIComponent(domain.trim())}`
    : `${BASE_URL}/available`;
  const res = await fetch(url);
  return res.json();
}
```

No other functions in this file change.

### Task 3: Frontend UI — Domain chip filter in Dashboard.jsx

**File**: `frontend/src/pages/Dashboard.jsx`

**What**: Add a `domainFilter` state variable, derive unique domain tags from the loaded `guides` list, render clickable chip buttons above the guides grid, and re-fetch guides from the API when a chip is clicked. The displayed guide cards should reflect the API-filtered result (i.e., `guides` state is what the API returns for the active filter — no client-side filtering needed).

**How**:

**Step A — Add state variable** (insert after line 109, inside the `useState` declarations at the top of the `Dashboard` component):

```js
const [domainFilter, setDomainFilter] = useState('');
```

**Step B — Add a handler function** (insert after the `handleSelectGuide` function, around line 260, before the return statement):

```js
const handleDomainFilter = async (domain) => {
  const next = domain === domainFilter ? '' : domain; // clicking active chip clears it
  setDomainFilter(next);
  try {
    const result = await getAvailableGuides(next || undefined);
    setGuides(Array.isArray(result) ? result : []);
  } catch {
    // leave guides as-is on network error
  }
};
```

**Step C — Add chip UI** (insert inside the `activeTab === 'Available Guides' && isStudent` block, between the header `<div>` that shows "Available Faculty Guides (N)" and the empty-state / grid conditional).

Find line 446 (closing `</div>` of the header row that contains the count and `prefMsg`). After that closing `</div>` and before the `{guides.length === 0 ? (` line, insert:

{/* Domain filter chips */}
{(() => {
  // Collect unique domain tags from the full (unfiltered) guides list.
  // Because the API returns only filtered results when a chip is active,

        </button>
      ))}

  );
})()}
